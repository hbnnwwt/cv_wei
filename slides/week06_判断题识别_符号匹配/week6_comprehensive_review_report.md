## 三、必须修复的问题（P0优先级）

### 1. AI编程辅助机制缺失
**影响**: 违背课程核心理念，无法体现"AI辅助编程"特色

**修复方案**:
```latex
% 在Live Coding模块前添加
\begin{frame}{AI辅助编程指南}
    \textbf{本周AI辅助任务：}
    \begin{itemize}
        \item 用AI解释圆度和凸性的数学原理
        \item 用AI生成特征提取代码框架
        \item 用AI调试特征阈值参数
    \end{itemize}

    \textbf{推荐Prompt模板（RTF框架）：}
    \begin{block}{示例}
    \texttt{[角色] 你是OpenCV专家 \\
    [任务] 解释Hu矩的物理意义 \\
    [格式] 用通俗语言+图示说明}
    \end{block}
\end{frame}
```

### 2. 分组原则完全缺失
**影响**: 学生无法有效分组协作，影响课堂组织

**修复方案**:
```latex
% 在00_logistics.tex中添加
\begin{frame}{分组与角色}
    \textbf{分组原则：}
    \begin{itemize}
        \item 每组4人，不同专业背景混合
        \item 确保每组至少有一名有编程基础的同学
    \end{itemize}

    \textbf{角色分工：}
    \begin{description}
        \item[组长] 协调进度，分配任务
        \item[技术负责人] 把关代码质量
        \item[模块开发A] 负责特征提取模块
        \item[模块开发B] 负责分类识别模块
    \end{description}

    \textbf{参与模式：}
    \begin{itemize}
        \item \textbf{观察者}: 理解原理，看演示
        \item \textbf{使用者}: 运行代码，调整参数
        \item \textbf{创造者}: 修改算法，创新功能
    \end{itemize}
\end{frame}
```

### 3. 多屏协同设计缺失
**影响**: 未体现"动静结合"的教学策略

**修复方案**:
```latex
% 在Live Coding前添加
\begin{frame}{演示模式说明}
    \textbf{双屏协作：}
    \begin{itemize}
        \item \textbf{主屏}: 显示代码和讲解
        \item \textbf{侧屏}: 实时运行效果和可视化
    \end{itemize}

    \textbf{AI辅助演示流程：}
    \begin{enumerate}
        \item 教师展示问题
        \item 用AI生成代码框架
        \item 师生共同完善代码
        \item 实时运行验证效果
    \end{enumerate}
\end{frame}
```
