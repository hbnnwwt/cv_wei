### 2. 核心理论与原理 (12-18页)

#### 2.1 Prompt工程基础 (4-5页)

**2.1.1 什么是Prompt？**
- 定义：给AI的指令或提示词
- 好Prompt的标准：具体明确、有上下文、有约束
- 模糊vs具体对比示例

**2.1.2 Prompt三要素**
- 角色（Role）：你是一位有5年OpenCV经验的工程师
- 任务（Task）：用Python和OpenCV实现人脸检测
- 约束（Constraints）：代码有详细注释、包含使用示例

**2.1.3 Prompt结构模板**
```markdown
角色：[身份设定]
任务：[具体描述]
输入：[数据格式]
输出：[期望结果]
要求：[具体约束]
```

#### 2.2 不同类型Prompt对比 (3-4页)

**2.2.1 代码生成Prompt**
- 功能描述
- 输入/输出说明
- 代码要求
- 示例代码

**2.2.2 代码解释Prompt**
- 粘贴代码
- 逐行解释要求
- 关键参数说明
- 可能的错误用法

**2.2.3 调试求助Prompt**
- 粘贴代码
- 错误信息
- 预期行为
- 上下文说明

**2.2.4 代码优化Prompt**
- 原始代码
- 优化目标（性能/可读性）
- 约束条件

#### 2.3 Prompt优化技巧 (3-4页)

**2.3.1 少样本提示（Few-Shot）**
- 提供1-3个示例
- 模仿学习

### 4. Live Coding实战 (20-25页) ⭐ 核心

#### 4.1 AI辅助实现人脸检测 (5-6页)

**4.1.1 阶段1：生成基础代码（10分钟）**
```
Prompt：
请用Python和OpenCV实现一个人脸检测程序：

功能：从图片中检测所有人脸，并用矩形框标注
输入：图片文件路径
输出：标注了人脸框的图片
要求：
- 使用OpenCV的Haar级联分类器
- 在每个人脸周围绘制绿色矩形框
- 显示检测到的人脸数量
- 代码有详细中文注释

**4.1.3 阶段3：批量处理（5分钟）**

#### 5.2 常见错误分析与解决 (2-3页)

**错误类型1：Prompt太模糊**
```
问题Prompt：
"帮我写代码"

分析原因：
- 缺少角色设定
- 缺少任务描述
- 缺少约束条件

解决方案：
- 添加角色
- 明确任务
- 增加约束
```

**错误类型2：Prompt太复杂**
```
问题Prompt：
"你是一位有5年OpenCV经验、精通Python和C++、熟悉深度学习和机器学习的工程师，请用Python和OpenCV实现一个完整的智能阅卷系统，包括图像采集、预处理、版面分析、OCR识别、评分输出等所有模块，要求代码有详细注释、性能优化、异常处理、单元测试、文档生成..."

分析原因：
- Prompt过长
- 信息过载
- AI容易混淆

解决方案：
- 拆分为多个子任务
- 逐步实现
- 逐步优化
```
